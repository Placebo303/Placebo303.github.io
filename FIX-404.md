# 🔧 修复 404 错误

## 问题原因

404 错误是因为：
1. **缺少 index.html** - 已修复 ✅
2. **文件未推送到 GitHub** - 需要你操作
3. **GitHub Pages 配置不正确** - 需要你操作

---

## 立即修复步骤

### 步骤 1：推送代码到 GitHub

打开 PowerShell，运行：

```powershell
cd D:\Code\academic-website
.\deploy.ps1
```

或者手动运行：

```powershell
cd D:\Code\academic-website
git init
git remote add origin https://github.com/Placebo303/Placebo303.github.io.git
git add .
git commit -m "Fix: Add index.html and complete website"
git push -u origin main --force
```

### 步骤 2：配置 GitHub Pages

1. 打开 https://github.com/Placebo303/Placebo303.github.io
2. 点击 **Settings**
3. 左侧菜单选择 **Pages**
4. 设置：
   - Source: **Deploy from a branch**
   - Branch: **main**
   - Folder: **/ (root)**
5. 点击 **Save**

### 步骤 3：等待部署

- 等待 1-5 分钟
- 查看 Actions 页面确认构建成功
- 访问 https://placebo303.github.io

## 验证清单

- [ ] 代码已推送到 GitHub
- [ ] GitHub Pages 设置为 main 分支
- [ ] 构建成功（Actions 页面显示绿色）

## 如果仍然 404

### 检查 1：确认分支名称

```powershell
git branch
```

应该显示 `* main`。如果是 `master`，运行：

```powershell
git branch -M main
git push -u origin main --force
```

### 检查 2：确认文件存在

访问 https://github.com/Placebo303/Placebo303.github.io，确认能看到 `index.html` 文件。

### 检查 3：查看构建日志

1. 打开 https://github.com/Placebo303/Placebo303.github.io/actions
2. 查看最新的构建记录
3. 如果有错误，根据错误信息修复
